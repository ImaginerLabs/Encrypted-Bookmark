# Encrypted Bookmark - 快速使用指南

**适用版本**：V1.3  
**最后更新**：2026-04-03

---

## 一、构建与安装

### 1.1 环境准备

确保已安装以下工具：
- **Node.js** >= 18（推荐 LTS 版本）
- **npm** >= 9

### 1.2 构建项目

```bash
# 克隆项目
git clone https://github.com/alex/Encrypted-Bookmark.git
cd Encrypted-Bookmark

# 安装依赖
npm install

# 构建生产版本
npm run build
```

构建完成后，`dist/` 目录即为可加载的插件包。

### 1.3 加载到 Chrome / Edge

1. 打开浏览器，访问 `chrome://extensions/`（Edge 为 `edge://extensions/`）
2. 开启右上角 **「开发者模式」** 开关
3. 点击 **「加载已解压的扩展程序」**
4. 选择项目的 `dist` 目录
5. 确认插件卡片显示 **「Encrypted Bookmark」** 且状态为已启用

> 💡 **提示**：安装成功后，浏览器工具栏会出现插件图标 🔐

## 二、首次使用

### 2.1 设置主密码

1. 点击浏览器工具栏的 **Encrypted Bookmark** 图标
2. 首次打开会显示密码设置界面
3. 输入主密码（要求 8-32 位字符）
4. 确认密码后点击 **「设置密码」**
5. 设置成功后自动进入书签管理界面

> ⚠️ **重要**：请牢记主密码！密码丢失后无法恢复数据。建议使用包含大小写字母、数字和特殊字符的强密码。

### 2.2 解锁插件

每次打开插件或浏览器空闲超过设定时间后（默认 15 分钟，可在设置页面修改），需要输入主密码解锁：

1. 点击插件图标
2. 在品牌化解锁页面输入主密码
3. 点击 **「解锁」**

> 🔒 **安全机制**：连续输错 3 次密码将锁定 30 秒，5 次将锁定 5 分钟。

## 三、核心功能

### 3.1 添加书签

**方式一：手动添加**
1. 在插件弹窗中点击 **「+」** 按钮
2. 输入书签标题和 URL
3. 选择所属文件夹（可选）
4. 添加标签（可选）
5. 点击 **「保存」**

**方式二：快速收藏当前页面**
1. 在任意网页上点击插件图标
2. 插件会自动获取当前页面的标题和 URL
3. 确认信息后点击 **「添加书签」**

**标记为"稍后再读"**：
- 在添加书签面板中，备注字段下方有 **「🕐 稍后再读」** Toggle 开关
- 开启开关后保存书签，该书签将被标记为"稍后再读"
- 开关默认关闭，不影响已有的添加书签流程

### 3.2 管理文件夹

**查看文件夹**：左侧侧边栏默认显示文件夹列表，点击文件夹可筛选该文件夹下的书签。

**新建文件夹**：
1. 在文件夹列表底部点击 **「+ 新建文件夹」**
2. 在出现的输入框中输入文件夹名称
3. 按 **Enter** 确认创建

**重命名文件夹**：
1. 右键点击要重命名的文件夹
2. 在弹出菜单中选择 **「重命名」**
3. 文件夹名称变为可编辑状态，修改后按 **Enter** 确认，或按 **Escape** 取消

**删除文件夹**：
1. 右键点击要删除的文件夹
2. 在弹出菜单中选择 **「删除」**
3. 确认弹窗会显示该文件夹下的书签数量
4. 确认删除后，文件夹下的书签将自动迁移至「未分类」

> ⚠️ **注意**：默认的「未分类」文件夹不可重命名或删除。

### 3.3 标签管理

**切换到标签视图**：
- 在左侧侧边栏顶部点击 **「🏷️ 标签」** Tab 即可切换到标签视图
- 点击 **「📁 文件夹」** Tab 可切回文件夹视图

**查看标签**：
- 标签列表显示所有标签，每个标签右侧显示使用次数（关联的书签数量）
- 点击标签可筛选该标签下的书签

**删除标签**：
1. 右键点击要删除的标签
2. 在弹出菜单中选择 **「删除」**
3. 确认弹窗会显示受影响的书签数量
4. 确认删除后，该标签将从所有关联书签中移除

**为书签添加标签**：
- 在添加或编辑书签时，可选择已有标签或输入新标签名称

### 3.4 搜索书签

在搜索框中输入关键词，支持：
- 按书签标题搜索
- 按 URL 搜索
- 按标签搜索

### 3.5 导入导出

进入 **设置页面**（右键插件图标 → 选项）：

**导入书签**：
1. 点击 **「导入」** 标签页
2. 选择文件（支持 Chrome HTML 书签文件 或 JSON 格式）
3. 选择导入策略（合并 / 覆盖）
4. 点击 **「开始导入」**

**导出书签**：
1. 点击 **「导出」** 标签页
2. 选择导出格式（JSON 明文 / JSON 加密）
3. 点击 **「开始导出」**
4. 文件将自动下载

## 四、设置页面

右键插件图标 → **「选项」** 进入设置页面：

| 功能 | 说明 |
|------|------|
| 修改密码 | 输入当前密码和新密码进行修改 |
| 存储方式 | 切换插件存储 / 本地文件存储 |
| 本地路径配置 | 配置本地文件存储路径 |
| 自动锁定时间 | 设置空闲自动锁定的时间（0/5/15/30/60 分钟，修改后实时生效，无需重启浏览器） |
| 导入导出 | 书签数据的导入和导出 |
| 重置数据 | ⚠️ 清除所有数据（不可恢复） |

## 五、存储方式说明

### 方式一：插件存储（默认）
- 使用 `chrome.storage.local` 存储
- 无需额外配置，开箱即用
- 容量限制约 10MB（约 3000 条书签）

### 方式二：本地文件存储
- 数据存储在本地文件系统
- 无容量限制
- 需要在设置页面配置存储路径
- 适合书签量较大的用户

## 六、常见问题

### Q: 忘记主密码怎么办？
**A**: 由于采用端到端加密，密码丢失后无法恢复数据。建议：
- 定期使用导出功能备份书签
- 如确需重置，可在设置页面点击「重置所有数据」重新开始

### Q: 插件图标无法显示？
**A**: 这可能是图标资源问题，不影响功能使用。点击图标仍可正常打开插件。

### Q: 提示「清单文件缺失或不可读」？
**A**: 加载插件时请确保选择的是 `dist` 目录，而非项目根目录。

### Q: 修改代码后如何更新插件？
**A**: 
1. 重新运行 `npm run build`
2. 在 `chrome://extensions/` 页面点击插件的刷新按钮

### Q: 支持哪些浏览器？
**A**: 支持所有基于 Chromium 的浏览器：
- Google Chrome 88+
- Microsoft Edge 88+
- Brave、Opera、Vivaldi 等

### Q: 数据安全吗？
**A**: 
- 所有书签数据使用 AES-256-GCM 加密存储
- 密钥通过 PBKDF2（100,000 次迭代）从密码派生
- 数据完全存储在本地，不上传任何服务器
- 密钥仅存在内存中，锁定后立即清除

## 七、开发模式

如需进行开发调试：

```bash
# 启动开发服务（支持热重载）
npm run dev

# 类型检查
npm run type-check

# 代码检查
npm run lint
```

### 调试方法
- **Popup 调试**：点击插件图标打开弹窗 → 右键弹窗 → 检查
- **Background 调试**：`chrome://extensions/` → 找到插件 → 点击「Service Worker」
- **Options 调试**：打开设置页面 → F12 打开 DevTools
