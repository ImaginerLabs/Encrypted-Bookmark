# Agent 协作流程详细说明

> 阶段间的上下文传递与回退机制详细规范。

## 整体流程图

```
用户需求
   │
   ▼
[阶段1] 产品经理 ──调研项目──▶ 输出 PRD 文档
   │                              │
   │                         PRD/{需求名称}.md
   │                              │
   ▼                              ▼
[阶段2] 前端开发 ───按PRD实现──▶ 代码 + E2E 测试
   │                              │
   │                              ▼
   ▼                         npm run test:e2e
[阶段3] 测试验证 ◀──运行测试────┘
   │        │
   │    通过 │  失败
   │        │   │
   │        │   └──▶ 回到阶段2修复（最多3次）
   │        │
   ▼        ▼
[阶段4] 文档沉淀 ───更新文档──▶ CHANGELOG + README + docs
   │
   ▼
 交付完成
```

## 各阶段 Task 调用参数

### 阶段 1 调用

```json
{
  "subagent_name": "通用产品经理",
  "description": "PRD 需求分析",
  "prompt": "<替换为用户需求 + 调研指令>",
  "mode": "default"
}
```

### 阶段 2 调用

```json
{
  "subagent_name": "通用资深前端研发",
  "description": "功能开发 + E2E测试",
  "prompt": "<替换为 PRD 路径 + 开发指令>",
  "mode": "acceptEdits"
}
```

### 阶段 3 调用

```json
{
  "subagent_name": "通用资深测试",
  "description": "E2E 测试验证",
  "prompt": "<替换为 PRD 路径 + 测试指令>",
  "mode": "default"
}
```

### 阶段 4 调用

```json
{
  "subagent_name": "通用资深前端研发",
  "description": "文档更新",
  "prompt": "<替换为 PRD 路径 + 文档指令>",
  "mode": "acceptEdits"
}
```

## 上下文传递规则

1. **PRD 路径**：阶段 1 产出的 `PRD/{需求名称}.md` 路径传递给后续所有阶段
2. **测试结果**：阶段 3 的测试结果（通过/失败/失败详情）传递给阶段 2（回退时）或阶段 4（通过时）
3. **需求摘要**：用户原始需求描述贯穿所有阶段

## 回退机制

```
测试失败
   │
   ▼
提取失败信息（用例名、错误信息、堆栈）
   │
   ▼
构造修复 prompt，包含：
  - PRD 路径
  - 失败用例列表
  - 错误详情
   │
   ▼
调用前端开发 Agent 修复
   │
   ▼
重新调用测试 Agent 验证
   │
   ├── 通过 → 进入阶段 4
   └── 失败 → 再次回退（最多 3 次）
           └── 3 次仍失败 → 暂停，向用户报告问题
```

## mode 选择说明

| 阶段 | mode | 理由 |
|------|------|------|
| 阶段 1（PRD） | `default` | 仅读取文件和创建 PRD 文档，无代码修改 |
| 阶段 2（开发） | `acceptEdits` | 需要修改源代码和 E2E 测试文件 |
| 阶段 3（测试） | `default` | 运行测试命令，不修改代码 |
| 阶段 4（文档） | `acceptEdits` | 需要修改文档文件 |
